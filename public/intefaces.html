Interfaces

An interface in Kotlin is a contract for classes that defines methods and properties a class must implement. It supports abstraction and multiple inheritance, making it ideal for shared behavior across unrelated classes.

Key Features of Interfaces

  1. Abstract and Default Implementations: Interfaces can declare abstract methods that must be implemented and default methods that provide a base implementation.

  2. Multiple Inheritance: Classes can implement multiple interfaces.

  3. Properties Without State: Interfaces can define properties, but they can't hold state—only provide default getters.

  4. No Constructors: Interfaces cannot have constructors.

interface Clickable { fun onClick() fun onLongClick() { println("Default long click behavior") } } class Button : Clickable { override fun onClick() { println("Button clicked!") } } fun main() { val button = Button() button.onClick() button.onLongClick() }

Use Cases for Interfaces

  1. Defining Behavior: Interfaces are ideal for defining shared behavior across unrelated classes.

  2. Multiple Inheritance: When a class needs to inherit from multiple sources of behavior.

  3. Contracts: Ensuring a class adheres to specific methods or properties.

Defining Abstract and Default Methods

Abstract methods

interface Shape { fun calculateArea(): Double }

Default implementations

interface Draggable { fun drag() { println("Dragging the item") } }

Properties in Interfaces

interface Vehicle { val maxSpeed: Int get() = 120 // Default getter fun drive() }

Multiple Inheritance in Interfaces

A class can implement multiple interfaces, combining their behavior

interface A { fun show() = println("A") } interface B { fun show() = println("B") } class C : A, B { override fun show() { super<A>.show() super<B>.show() } } fun main() { val c = C() c.show() }

Sealed Interface

Since Kotlin 1.5, interfaces can be sealed, allowing you to restrict their implementations to the same file:

sealed interface Operation { data class Add(val value: Int) : Operation data class Subtract(val value: Int) : Operation object NoOperation : Operation }

Use case:

fun handleOperation(operation: Operation) { when (operation) { is Operation.Add -> println("Adding ${operation.value}") is Operation.Subtract -> println("Subtracting ${operation.value}") Operation.NoOperation -> println("No operation performed") } }

Benefits of Interfaces

  • Flexible Design: Allows multiple inheritance and shared behavior across unrelated classes.

  • Modular Design: Promotes clean and modular design patterns.

  • Default Behavior: Enables reusable default method implementations.

When to Use Interfaces

  1. When you need multiple inheritance.

  2. When defining contracts or shared behavior for unrelated classes.

  3. For designing plugin-like extensibility.

Limitations of Interfaces

  • No Constructors: Interfaces cannot have constructors.

  • No State: Interfaces cannot hold fields, only properties with default getters.

  • Overriding Conflicts: Classes implementing multiple interfaces with the same method must explicitly resolve conflicts.
