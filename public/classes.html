Classes

In Kotlin, classes are blueprints for creating objects, encapsulating data and behavior. They are central to object-oriented programming and are used to model real-world entities.

Constructor

In Kotlin, constructors are special functions used to initialize objects. There are two types of constructors:

Primary Constructor

  • The main constructor of a class, defined as part of the class header.

  • Used for straightforward initialization of properties.

  • Does not contain a body unless combined with init blocks for additional logic.

class Person(val name: String, var age: Int) { fun greet() { println("Hello, my name is $name and I am $age years old.") } } val person = Person("Alice", 30) person.greet() // Output: Hello, my name is Alice and I am 30 years old.

Usage

  1. The parameters of the primary constructor can be directly used to initialize class properties.

  2. It is concise and typically sufficient for simple classes.

Secondary Constructor

  • An additional constructor used when you need alternative ways to create an object.

  • Defined inside the class body using the constructor keyword.

  • Can include custom initialization logic.

  • Must delegate to the primary constructor (if it exists) using the this keyword.

class Rectangle(val width: Int, val height: Int) { var area: Int = 0 constructor(side: Int) : this(side, side) { area = side * side } } val square = Rectangle(5) println(square.area) // Output: 25

When to Use Each?

  1. Use Primary Constructor for the main or default initialization logic.

  2. Use Secondary Constructor for:

    • Alternate ways to initialize an object.

    • Handling optional parameters or different initialization paths.

class Student(val name: String) { var grade: Int = 0 // Secondary constructor constructor(name: String, grade: Int) : this(name) { this.grade = grade } } val student1 = Student("Alice") val student2 = Student("Bob", 90) println("${student1.name}, Grade: ${student1.grade}") // Alice, Grade: 0 println("${student2.name}, Grade: ${student2.grade}") // Bob, Grade: 90

Init Block

Characteristics of init Block

  • Runs Automatically: The init block is called automatically whenever an instance of the class is created.

  • Order of Execution: If there are multiple init blocks, they are executed in the order they are declared in the class. The init blocks run before the body of the primary constructor.

  • Access to Constructor Parameters: The init block has access to the parameters of the primary constructor.

  • No Explicit Call Needed: Unlike functions, you do not need to explicitly call the init block.

  • Optional:

  • You can omit the init block if no initialization logic is required.

class Circle(val radius: Double) { val area: Double init { area = Math.PI * radius * radius } }

Types of Classes

Regular Clas

A standard class for creating objects.

class Animal(val name: String, val age: Int)

Data Class

Used to hold data. Automatically provides toString(), equals(), hashCode(), and copy() methods.

data class User(val id: Int, val name: String)

Abstract Class

Cannot be instantiated directly. Used as a base class.

abstract class Vehicle { abstract fun drive() }

Open Class

Allows other classes to inherit from it (default classes are final in Kotlin).

open class Shape class Circle : Shape()

Sealed Class

Used to represent a restricted hierarchy of types.

sealed class Result data class Success(val data: String) : Result() data class Error(val message: String) : Result()

Enum Class

Defines a set of constants.

enum class Direction { NORTH, SOUTH, EAST, WEST }

Companion Object

Allows defining static-like members for a class.

class MathUtils { companion object { fun add(a: Int, b: Int) = a + b } } val result = MathUtils.add(3, 5) println(result) // Output: 8

Getters And Setters

In Kotlin, getters and setters are special methods used to encapsulate the fields of a class, allowing controlled access and modification of a property. Unlike Java, Kotlin provides a concise and idiomatic way of defining and customizing these methods for properties.

In Kotlin, every property has a default getter and, if it's var, a default setter. These allow you to retrieve and modify the value of the property.

Default Behavior

  • val (Immutable property): Only a getter is generated.

  • var (Mutable property): Both a getter and a setter are generated.

class Person { var name: String = "John Doe" // Default getter and setter are generated val age: Int = 25 // Only a default getter is generated } fun main() { val person = Person() println(person.name) // Calls the default getter person.name = "Jane Doe" // Calls the default setter println(person.age) // Calls the default getter }

Custom Getters and Setters

You can customize the behavior of getters and setters by overriding their default implementation.

Custom Getter A custom getter allows you to define how a property's value is retrieved

class Circle(private val radius: Double) { val area: Double get() = Math.PI * radius * radius // Calculate the area dynamically } fun main() { val circle = Circle(5.0) println(circle.area) // Calls the custom getter }

Custom Setter A custom setter allows you to control how a property’s value is set

class User { var password: String = "" set(value) { if (value.length >= 8) { field = value // `field` represents the backing field } else { println("Password must be at least 8 characters long") } } } fun main() { val user = User() user.password = "short" // Prints warning user.password = "longenough" // Sets the password }

Backing Fields and Backing Properties

Backing Field (field)

  • Kotlin uses a backing field to store the actual value of a property.

  • The field keyword is only accessible in custom getters and setters

class Box { var width: Int = 0 set(value) { field = if (value >= 0) value else 0 // Use `field` to assign the value } }

Backing Property

Instead of using a backing field, you can use another private property to hold the data.

class Rectangle { private var _height: Int = 0 var height: Int get() = _height set(value) { _height = if (value >= 0) value else 0 } }

Properties in Primary Constructor

You can define properties directly in the primary constructor with default getters and setters.

class Person(var name: String, val age: Int)

If you need custom logic, you can add a custom getter or setter:

class Person(var name: String, val birthYear: Int) { val age: Int get() = 2024 - birthYear // Custom getter }

Visibility Modifiers for Getters and Setters

Getters and setters can have different visibility modifiers, allowing controlled access.

class Account { var balance: Int = 0 private set // Setter is private } fun main() { val account = Account() println(account.balance) // Allowed (getter is public) // account.balance = 100 // Error: Cannot access private setter }

Nested and Inner Classes

Nested Class

Does not have access to the outer class members.

class Outer { class Nested { fun message() = "Nested class" } } val nested = Outer.Nested().message() println(nested) // Output: Nested class

Inner Class

Has access to the outer class members.

class Outer { private val message = "Inner class" inner class Inner { fun display() = message } } val inner = Outer().Inner().display() println(inner) // Output: Inner class
