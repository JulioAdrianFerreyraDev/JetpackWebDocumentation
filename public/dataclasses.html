Data Classes

Data classes in Kotlin are specifically designed to hold data. They provide a concise way to declare classes that are primarily used to store and manage information, with automatic implementations of commonly used methods like equals(), hashCode(), toString(), and copy().

Defining a Data Class

A data class is declared using the data keyword before the class keyword.

data class User(val id: Int, val name: String)

Key Features:

  1. Automatically generates: toString() Provides a string representation of the object.

val user = User(1, "Alice") println(user) // Output: User(id=1, name=Alice)

equals() and hashCode() Used for comparison and hashing. Two data class objects are equal if their properties are equal.

val updatedUser = user1.copy(name = "Bob") println(updatedUser) // Output: User(id=1, name=Bob)

copy() Creates a new object by copying the existing one and optionally changing some properties

val user = User(1, "Alice") println(user) // Output: User(id=1, name=Alice)

  1. Primary constructor must have at least one parameter.

  2. Parameters in the primary constructor must be declared as val or var.

Example

data class Book(val title: String, val author: String, val price: Double) fun main() { val book = Book("Kotlin in Action", "Dmitry Jemerov", 35.99) println(book) // Output: Book(title=Kotlin in Action, author=Dmitry Jemerov, price=35.99) val anotherBook = book.copy(price = 29.99) println(anotherBook) // Output: Book(title=Kotlin in Action, author=Dmitry Jemerov, price=29.99) }

Requirements for Data Classes

  • The primary constructor must have at least one parameter.

  • All parameters must be declared as val or var.

  • Data classes cannot be abstract, open, sealed, or inner.

Note

You can define default values for properties in the primary constructor.

Customizing Data Classes

You can override the default behavior of the generated methods.

data class Person(val name: String, val age: Int) { override fun toString(): String { return "Person(name=$name, age=$age)" } }

Nested and Inherited Data Classes

Nested Data Classes

Data classes can contain other data classes

data class Address(val city: String, val country: String) data class Employee(val id: Int, val name: String, val address: Address) val address = Address("New York", "USA") val employee = Employee(1, "John", address) println(employee) // Output: Employee(id=1, name=John, address=Address(city=New York, country=USA))

Inheritance

While data classes cannot inherit other classes directly, they can implement interfaces.

interface Identifiable { val id: Int } data class Product(override val id: Int, val name: String) : Identifiable

Use Cases of Data Classes

  • Data Transfer Objects (DTOs): Pass data between different layers of an application.

  • Immutable Objects: Represent objects that should not change after creation.

  • Mapping and Transformations: Use destructuring and mapping for concise transformations.

  • Collections: Store data in collections like lists, maps, or sets.

Limitations

  • Cannot be used for classes requiring inheritance.

  • Generated methods might not be suitable for complex objects (customization may be needed).

  • Large data classes can lead to verbose toString() outputs.
