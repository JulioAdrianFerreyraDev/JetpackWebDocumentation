Returns and Jumps

return

The return keyword is used to exit a function and optionally send a value back to the calling function. In Kotlin, it’s commonly used in two contexts: within regular functions and within lambda expressions.

return // without a value, only valid in functions with no specified return type return value // used to return a specific value

Use in Functions

In a function, return immediately exits the function and can return a value if specified. The return type of the function must match the type of the value being returned (if any).

fun add(a: Int, b: Int): Int { return a + b }

Use in Lambdas

In Kotlin, return can be a bit different within lambdas, where it is used with labeled returns (return@label). A non-labeled return within a lambda will exit the containing function.

fun checkNumbers(numbers: Lis<Int>): List<Int> { return numbers.filter { if (it < 0) return@filter false // returns from the lambda, not from checkNumbers true } }

Kotlin offers return@label for local returns within lambdas, helping you control the return scope.

Continue

The continue keyword is used within loops (for, while, do-while) to skip the current iteration and proceed to the next iteration.

continue

Continue Use with Loops

When continue is encountered within a loop, the code following continue is skipped, and the next iteration begins. It is often used with conditions to control iteration flow.

for (i in 1..5) { if (i == 3) continue // skips when i is 3 println(i) }

Labeled Continue

In nested loops, continue can be combined with a label to specify which loop should continue its next iteration.

outer@ for (i in 1..3) { for (j in 1..3) { if (i == 2 && j == 2) continue@outer println("i: $i, j: $j") } }

In this example, continue@outer skips directly to the next iteration of the outer loop, bypassing the inner loop.

Break

The break keyword is used to exit a loop (either for, while, or do-while) prematurely, stopping further iterations.

break

Break Use with Loops

When break is used in a loop, it immediately terminates the loop and proceeds with the code following the loop.

for (i in 1..5) { if (i == 3) break // exits loop when i is 3 println(i) }

Labeled Break

Like continue, break can also be used with labels to exit from a specific loop in nested structures.

outer@ for (i in 1..3) { for (j in 1..3) { if (i == 2 && j == 2) break@outer println("i: $i, j: $j") } }

In this case, break@outer exits the outer loop completely when i is 2 and j is 2, ignoring any remaining iterations in both loops.
