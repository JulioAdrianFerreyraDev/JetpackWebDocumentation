Enum Classes

Kotlin provides a special type of class called Enum Class to define a set of constants. These constants are objects that are used to represent a predefined list of possible values. Enum classes are ideal for use cases like representing states, directions, or predefined options in a type-safe manner.

An enum class in Kotlin is a special class that represents a group of constants. These constants are instances of the enum class itself and are implicitly declared as static and final.

Declaring an Enum Class

Basic Declaration

enum class Direction { NORTH, SOUTH, EAST, WEST }

Here, NORTH, SOUTH, EAST, and WEST are the constants of the Direction enum.

Usage

fun getDirectionMessage(direction: Direction): String { return when (direction) { Direction.NORTH -> "You are heading north." Direction.SOUTH -> "You are heading south." Direction.EAST -> "You are heading east." Direction.WEST -> "You are heading west." } } val direction = Direction.NORTH println(getDirectionMessage(direction)) // Output: You are heading north.

Properties and Methods of Enum Classes

Each enum constant has a few predefined properties and methods:

  • name: The name of the enum constant as a string.

  • ordinal: The position of the enum constant (zero-based index).

  • values(): Returns an array of all enum constants.

  • valueOf(name: String): Returns the enum constant matching the given name

enum class Day { MONDAY, TUESDAY, WEDNESDAY } fun main() { val day = Day.MONDAY println(day.name) // Output: MONDAY println(day.ordinal) // Output: 0 val days = Day.values() println(days.joinToString()) // Output: MONDAY, TUESDAY, WEDNESDAY val tuesday = Day.valueOf("TUESDAY") println(tuesday) // Output: TUESDAY }

Adding Custom Properties and Methods

Enum constants can have properties:

enum class Planet(val mass: Double, val radius: Double) { EARTH(5.972e24, 6371.0), MARS(6.39e23, 3389.5); fun surfaceGravity(): Double { val G = 6.67430e-11 return G * mass / (radius * radius) } }

Usage

fun main() { val gravity = Planet.EARTH.surfaceGravity() println("Earth's surface gravity: $gravity") }

Enum Constants as Objects

Each constant in an enum class is an object, and you can override methods or properties for individual constants.

enum class AnimalSound { DOG { override fun sound() = "Woof" }, CAT { override fun sound() = "Meow" }; abstract fun sound(): String } fun main() { println(AnimalSound.DOG.sound()) // Output: Woof println(AnimalSound.CAT.sound()) // Output: Meow }

Enum with Interfaces

Enum classes can implement interfaces.

interface DisplayName { fun displayName(): String } enum class Color : DisplayName { RED { override fun displayName() = "Red Color" }, BLUE { override fun displayName() = "Blue Color" } }

Common Use Cases

  • State Management: Representing states in an application (e.g., Loading, Success, Error).

  • Direction: Representing directions or navigation options.

  • Error Codes: Predefined error types or status codes.

  • Configuration Options: Predefined settings or modes.

Limitations of Enum Classes

  • Immutability: Enum constants are immutable.

  • Singleton Nature: Enum constants are single instances; they can't be instantiated multiple times.

  • Lack of Inheritance: Enum classes cannot extend other classes but can implement interfaces.

Enum class vs sealed class

Both enum class and sealed class are powerful constructs in Kotlin, but they serve different purposes and have distinct characteristics. Here’s a detailed comparison to help you understand their differences, use cases, and benefits.

Enum Class

  • Used to represent a fixed set of constants.

  • Each constant is a singleton instance of the enum class.

  • Typically used for simple categorizations or states with predefined values.

Sealed Class

  • Used to represent a restricted class hierarchy.

  • Allows more complex state modeling with multiple subclasses.

  • Useful when each type needs additional data or specific behavior.

Use Cases

Enum Class Use Cases

  • State Management: Represent simple states (e.g., LOADING, SUCCESS, ERROR)

enum class Status { LOADING, SUCCESS, ERROR }

  • Fixed Categories: Days of the week, directions, colors, etc

  • Switch Statements: Easy to use in when expressions without extra logic

Sealed Class Use Cases

  • Complex States: Model states with different data for each state

sealed class Result { object Loading : Result() data class Success(val data: String) : Result() data class Error(val message: String) : Result() }

  • Hierarchical Models: Model hierarchical relationships with a fixed set of subclasses.

sealed class Shape { data class Circle(val radius: Double) : Shape() data class Rectangle(val width: Double, val height: Double) : Shape() }

  • Rich Behaviors: Provide methods or logic specific to each subclass.

Matching with when

Enum

The when block automatically covers all enum values. If a value is missing, the compiler will give a warning.

enum class Status { LOADING, SUCCESS, ERROR } fun getMessage(status: Status): String { return when (status) { Status.LOADING -> "Loading..." Status.SUCCESS -> "Success!" Status.ERROR -> "Error occurred." } }

Sealed Class The when block must explicitly cover all subclasses or include an else branch.

sealed class Result { object Loading : Result() data class Success(val data: String) : Result() data class Error(val message: String) : Result() } fun getMessage(result: Result): String { return when (result) { is Result.Loading -> "Loading..." is Result.Success -> "Success: ${result.data}" is Result.Error -> "Error: ${result.message}" } }

Advantages and Limitations

Enum Class Advantages

  • Simple Syntax: Easy to define and use.

  • Type-Safe Constants: Provides predefined, type-safe constants.

  • Built-in Utility Methods: Includes values(), valueOf(), ordinal, and name. Enum Class Limitations

  • Cannot hold different data per constant.

  • Limited to static values without dynamic behavior.
