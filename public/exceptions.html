Exceptions

Exception handling in Kotlin is similar to Java but offers more concise syntax and modern features. It allows developers to manage runtime errors in a controlled way, avoiding crashes and enabling smoother user experiences.

Basics of Exception Handling

In Kotlin, exceptions are handled using the try, catch, finally, and throw keywords.

try { // Code that may throw an exception } catch (e: ExceptionType) { // Handle the exception } finally { // Code that will always run, regardless of an exception }

Key Points:

  1. try: Wraps the code block where an exception might occur.

  2. catch: Catches specific types of exceptions.

  3. finally: Executes after try or catch blocks, whether an exception occurs or not.

  4. throw: Used to explicitly throw an exception.

throw Keyword

In Kotlin, throw is used to manually throw an exception. This is helpful for enforcing specific conditions.

fun validateAge(age: Int) { if (age < 18) { throw IllegalArgumentException("Age must be at least 18") } } try { validateAge(15) } catch (e: IllegalArgumentException) { println(e.message) } // Output: Age must be at least 18

Custom Exceptions

You can create custom exceptions by extending the Exception class or any of its subclasses. This helps in creating more descriptive exceptions that are specific to your application's logic.

class InvalidAgeException(message: String) : Exception(message) fun register(age: Int) { if (age < 18) throw InvalidAgeException("You must be 18 or older to register.") } /* Exception Invocation */ try { register(16) } catch (e: InvalidAgeException) { println(e.message) } // Output: You must be 18 or older to register.

Checked vs. Unchecked Exceptions

Kotlin doesn’t have checked exceptions, unlike Java. This means you aren’t required to handle exceptions in code, making the syntax cleaner and more flexible. You can still handle exceptions explicitly if needed, but there is no compiler-enforced requirement.

Using try as an Expression

In Kotlin, try is an expression, which means it can return a value. You can assign the result of a try-catch block to a variable.

val result = try { "Hello".toInt() } catch (e: NumberFormatException) { 0 // Default value if exception occurs } println(result) // Output: 0

Common Exception Types

  • ArithmeticException: Thrown when an arithmetic error, such as division by zero, occurs.

  • NullPointerException: Thrown when trying to use a null object reference.

  • NumberFormatException: Thrown when trying to convert an invalid string to a number.

  • IllegalArgumentException: Thrown when an invalid argument is passed to a function.

  • IllegalStateException: Thrown when a method is invoked at an inappropriate time.

Extension Functions and Null Safety

Kotlin’s null safety features allow for smoother handling of NullPointerException. Using ?. and ?: can help avoid these exceptions.

val nullableString: String? = null println(nullableString?.length ?: "No length") // Output: No length

runCatching

The runCatching function is an alternative to try-catch, allowing for cleaner code by capturing exceptions and managing success and failure cases.

val result = runCatching { "123".toInt() }.getOrElse { println("An error occurred: ${it.message}") 0 } println(result) // Output: 123 val failedResult = runCatching { "abc".toInt() }.getOrElse { println("An error occurred: ${it.message}") 0 } println(failedResult) // Output: An error occurred: For input string: "abc" // 0
