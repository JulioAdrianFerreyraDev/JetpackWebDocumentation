Strings and Numbers

In Kotlin, everything is an object in the sense that you can call member functions and properties on any variable. While certain types have an optimized internal representation as primitive values at runtime (such as numbers, characters, booleans and others), they appear and behave like regular classes to you.

String

Strings in Kotlin are represented by the type String

Generally, a string value is a sequence of characters in double quotes ("):

val stringValue : String = "This is a String"

Strings are immutable. Once you initialize a string, you can't change its value or assign a new value to it. All operations that transform strings return their results in a new String object, leaving the original string unchanged:

val str = "abcd" // Creates and prints a new String object println(str.uppercase()) // ABCD // The original string remains the same println(str)

Elements of a string are characters that you can access via the indexing operation: s[i]. You can iterate over these characters with a for loop:

for (c in str) { println(c) }

Escaped Strings

Escaped strings can contain escaped characters.

val s = "Hello, world!\n"

Multiline strings

Multiline strings can contain newlines and arbitrary text. It is delimited by a triple quote ("""), contains no escaping and can contain newlines and any other characters:

val text = """ for (c in "foo") print(c) """

Interpolation and Concatenation

The difference between interpolation and concatenation is that interpolation places a string (or another value) within another string, while concatenation joins two or more strings together, one after the other.

Interpolation

val language : String = "Kotlin" println("hello from $language")

If you need to evaluate a function or add a piece of code that isn't a string, enclose it within {} brackets.

todos = listOf("Mop the floor", "Make the bed", "Wash the dishes", "Take a shower") println("You have ${todos.size} todos")

You can also use interpolation to add the dollar sign "$"

val price = """ ${'$'}_9.99 """

Concatenation

To concatenate two or more strings, add the + operator between each string.

print("Bonjour " + "Hello " + "Konnichiwa " + "Hola ")

String Format

To format a string to your specific requirements, use the String.format() function.

The String.format() function accepts a format string and one or more arguments. The format string contains one placeholder (indicated by %) for a given argument, followed by format specifiers. Format specifiers are formatting instructions for the respective argument, consisting of flags, width, precision, and conversion type. Collectively, format specifiers shape the output's formatting. Common format specifiers include %d for integers, %f for floating-point numbers, and %s for strings. You can also use the argument_index$ syntax to reference the same argument multiple times within the format string in different formats.

// Formats an integer, adding leading zeroes to reach a length of seven characters val integerNumber = String.format("%07d", 31416) println(integerNumber) // 0031416 // Formats a floating-point number to display with a + sign and four decimal places val floatNumber = String.format("%+.4f", 3.141592) println(floatNumber) // +3.1416 // Formats two strings to uppercase, each taking one placeholder val helloString = String.format("%S %S", "hello", "world") println(helloString) // HELLO WORLD // Formats a negative number to be enclosed in parentheses, then repeats the same number in a different format (without parentheses) using `argument_index$`. val negativeNumberInParentheses = String.format("%(d means %1\$d", -31416) println(negativeNumberInParentheses) //(31416) means -31416

Int

In Kotlin, integers are represented by the Int class, which provides a series of methods and properties for basic arithmetic, comparison, conversion, and manipulation.

Properties:

  • MAX_VALUE: Largest value an Int can hold (2^31 - 1).

  • MIN_VALUE: Smallest value an Int can hold (-2^31).

  • absoluteValue: Returns the non-negative value of the integer.

  • sign: Returns -1, 0, or 1 depending on whether the integer is negative, zero, or positive.

Double

The Double type in Kotlin is a 64-bit floating-point number. It is the default type for decimal values in Kotlin and provides high precision, which is suitable for complex calculations and scientific purposes. Its range and precision are significantly higher than that of the Float type.

Double Properties

  • MAX_VALUE: The maximum finite positive value of a Double, approximately 1.7976931348623157E308.

  • MIN_VALUE: The minimum positive nonzero value of a Double, approximately 4.9E-324.

  • POSITIVE_INFINITY: Represents positive infinity (1.0 / 0.0).

  • NEGATIVE_INFINITY: Represents negative infinity (-1.0 / 0.0).

  • NaN: Represents "Not-a-Number" for invalid calculations (e.g., 0.0 / 0.0).

Float

The Float type in Kotlin is a 32-bit floating-point number that follows IEEE 754 standards for representing decimal values. Floats are typically used when you need a smaller range of decimal numbers, especially if memory efficiency is important. This type is distinct from Double, which is a 64-bit floating-point number.

Float Properties

  • MAX_VALUE: The largest finite value representable by a Float, which is approximately 3.4028235E38.

  • MIN_VALUE: The smallest positive nonzero value that a Float can represent, which is approximately 1.4E-45.

  • NEGATIVE_INFINITY: Represents negative infinity in Float values.

  • POSITIVE_INFINITY: Represents positive infinity in Float values.

  • NaN: Stands for "Not-a-Number," a value representing an undefined or unrepresentable result in floating-point calculations.

It can be represented using the Float or the f keyword:

val floatNumber : Float = 2.3f

Long

The Long type in Kotlin is a 64-bit integer, which provides a much larger range than Int. It is commonly used when working with large integer values that exceed the Int range. Like all integer types in Kotlin, Long does not support decimals and is used exclusively for whole numbers.

Properties of Long:

  • MAX_VALUE: The largest value that Long can represent (9,223,372,036,854,775,807).

  • MIN_VALUE: The smallest value that Long can represent (-9,223,372,036,854,775,808).

Long values can be declared using the capital letter L after any Integer:

val longNumber = 30L

Short

The Short type in Kotlin is a 16-bit integer, with a smaller range than Int and Long. It's useful for memory-efficient storage when only small values are required. The Short type does not support decimals and is used for whole numbers only

Short Properties

  • MAX_VALUE: The largest value that Short can represent (32,767).

  • MIN_VALUE: The smallest value that Short can represent (-32,768).

Byte

The Byte type in Kotlin is an 8-bit integer, with a range from -128 to 127. It's primarily used in applications where minimal memory usage is required or when working with binary data. Like other integer types, Byte does not support decimals.

Byte Properties

  • MIN_VALUE: The minimum value for Byte (-128).
