Generics

Generics in Kotlin allow you to write flexible and reusable code that can operate on different data types while ensuring type safety. They are particularly useful when working with collections or creating custom data structures.

Key Features of Generics

  • Type Safety: Prevents runtime type casting errors.

  • Reusability: Enables the use of a single implementation for different types.

  • Flexibility: Allows defining constraints on types.

Defining Generics

Generic Classes

You can declare a class with one or more type parameters.

class Box<T>(val value: T) fun main() { val intBox = Box(42) // Box<Int> val stringBox = Box("Hello") // Box<String> println(intBox.value) // 42 println(stringBox.value) // Hello }

Generic Functions

You can also define functions with type parameters.

fun <T> printItem(item: T) { println(item) } fun main() { printItem(42) // Prints: 42 printItem("Hello") // Prints: Hello }
