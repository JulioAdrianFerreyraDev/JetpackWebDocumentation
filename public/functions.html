Functions

Functions in Kotlin allow you to encapsulate reusable pieces of logic, making code modular, organized, and easier to understand. Kotlin functions are flexible, with support for default arguments, named parameters, higher-order functions, lambdas, and extension functions.

Defining Functions

Functions are defined using the fun keyword, followed by the function name, parameters (if any), and the return type.

fun functionName(parameters): ReturnType { // Function body }

Example

fun greet(name: String): String { return "Hello, $name!" }

This function takes a String parameter called name and returns a greeting message of type String.

Calling Functions

To call a function, use the function name followed by parentheses with arguments (if any).

val message = greet("Alice") println(message) // Output: Hello, Alice!

Function Parameters

You can declare a function with one or more parameters:

Single Parameter

fun square(number: Int): Int { return number * number }

Multiple Parameters

fun add(a: Int, b: Int): Int { return a + b }

Default Parameters

Kotlin allows you to assign default values to function parameters, which makes them optional when calling the function.

fun greet(name: String = "Guest"): String { return "Hello, $name!" } println(greet()) // Output: Hello, Guest! println(greet("Alice")) // Output: Hello, Alice!

Named Arguments

When calling a function, you can specify arguments by name, which improves readability.

fun calculateRectangleArea(length: Int, width: Int): Int { return length * width } val area = calculateRectangleArea(width = 5, length = 10)

Return Types

Functions can return a value, with the return type specified after the parameter list. If a function does not return a meaningful value, use Unit (similar to void in other languages), though Unit can be omitted as it’s the default.

fun printMessage(message: String): Unit { println(message) }

Higher-Order Functions

A higher-order function is a function that takes another function as a parameter or returns a function. In Kotlin, functions can be treated as data and passed around.

fun operateOnNumbers(a: Int, b: Int, operation: (Int, Int) -> Int): Int { return operation(a, b) } val sum = operateOnNumbers(3, 4) { x, y -> x + y } println(sum) // Output: 7

If the last parameter is a function of any type, you can call and define it as a function body

Inline Functions

To avoid the performance overhead of higher-order functions, Kotlin provides inline functions, which copy the function’s code directly at the call site.

inline fun inlineFunctionExample(action: () -> Unit) { println("Start") action() println("End") }

Lambda Expressions and Anonymous Functions

Kotlin supports lambdas, which are essentially anonymous functions (functions without names). Lambdas can be passed to higher-order functions.

val add = { a: Int, b: Int -> a + b } println(add(2, 3)) // Output: 5

Extension Functions

Kotlin allows you to add new functionality to existing classes without modifying their code. These are called extension functions.

fun String.addExclamation(): String { return this + "!" } println("Hello".addExclamation()) // Output: Hello!

Scope Functions

Kotlin provides several built-in functions (let, run, with, apply, and also) for working with objects in a functional way. They are commonly used to manipulate objects within a limited scope.

let

  • Purpose: Used to execute a block of code on an object and return the result of the block. Commonly used for null checks.

  • Context Object: it (implicit name of the object).

val name: String? = "Alice" name?.let { println("Hello, $it!") // Executes only if name is not null } // Output: Hello, Alice!

Use Case:

  • Safely accessing nullable objects.

  • Executing code on an object and returning a result.

run

  • Purpose: Executes a code block on an object, useful when you want to initialize or configure an object and return a result.

  • Context Object: this (can use without naming explicitly).

val greeting = "Hello".run { "$this, World!" // 'this' refers to the string "Hello" } println(greeting) // Output: Hello, World!

Use Case:

  • Initialize an object and produce a result.

  • Group configuration code that requires a return value.

with

  • Purpose: Similar to run, but it is not called on an object. Instead, it takes the object as a parameter, useful for grouping multiple operations on the same object.

  • Context Object: this

val person = Person("Alice", 25) val info = with(person) { "Name: $name, Age: $age" } println(info) // Output: Name: Alice, Age: 25

Use Case:

  • Performing multiple operations on an object.

  • Returning a result after working with an object.

apply

  • Purpose: Executes a block of code on an object and returns the same object. Useful for initializing or configuring an object.

  • Context Object: this

val person = Person().apply { name = "Alice" age = 25 } println(person.name) // Output: Alice

Use Case:

  • Configuring properties of an object.

  • Builder pattern where you configure an object and return it.

also

  • Purpose: Similar to apply, but the block does not return the object. Instead, it returns the object itself. Used for performing additional actions without changing the object.

  • Context Object: it

val numbers = mutableListOf(1, 2, 3) numbers.also { println("Original list: $it") }.add(4) println(numbers) // Output: [1, 2, 3, 4]

Use Case:

  • Logging or debugging information.

  • Running additional operations on an object without modifying it.

Recursive Functions

A recursive function is a function that calls itself. Kotlin supports recursive functions, but be cautious to include a base case to prevent infinite recursion.

fun factorial(n: Int): Int { return if (n <= 1) 1 else n * factorial(n - 1) } println(factorial(5)) // Output: 120

Tail-Recursive Functions

Kotlin supports tail-recursive functions, which optimize recursive calls to prevent stack overflow by reusing the same stack frame.

tailrec fun factorialTail(n: Int, acc: Int = 1): Int { return if (n <= 1) acc else factorialTail(n - 1, acc * n) } println(factorialTail(5)) // Output: 120

suspend Functions

The suspend keyword in Kotlin is used for functions that can be paused and resumed at a later time. They are primarily used with coroutines for asynchronous and non-blocking operations. Suspend functions allow you to write asynchronous code in a sequential style.

suspend fun fetchUserData(): User { // Simulated delay delay(1000L) return User("Alice", 25) }

Using a Suspend Function

To call a suspend function, you need to be in a coroutine or another suspend function.

fun main() = runBlocking { val user = fetchUserData() println("Fetched user: $user") }
