Jetpack Compose Help

Navigation

The Navigation component provides support for Jetpack Compose applications. You can navigate between composables while taking advantage of the Navigation component's infrastructure and features.

Implementation

The following implementation must be added into build.gradle dependencies:

dependencies { def nav_version = "2.7.7" implementation "androidx.navigation:navigation-compose:$nav_version" }
dependencies { val nav_version = "2.7.7" implementation("androidx.navigation:navigation-compose:$nav_version") }

Version warning

Be sure you are using the version you need to work in your application. The version used above is just an example and the latest version available in 06/07/2024

Getting Started

When implementing navigation in an app, implement a navigation host, graph, and controller. For more information, see the Navigation overview.

NavController

To create a NavController when using Jetpack Compose, call rememberNavController()

val navController = rememberNavController()

NavHost

After the navController declaration, you need to add a navHost that allows to create all the routes by adding a name to the composable destination.

NavHost(navController = navController, startDestination = "startDestinationView") { composable(route = "routeName") { ComposableView() } }

Navigation From a Composable

navController.navigate(route = "routeName")
navController.navigate(route = "routeName/?${parameter}") // The "?" only goes if the param is nullSafety

You can also navigate back into the graph navigation:

popUpTo(id = 0) // If 0 (zero) is given, then it closes the app, //otherwise it will navigate to the tab given graph navigation index navController.popBackStack() // Navigate back to the previous screen

Expose events from your composables

When a composable function needs to navigate to a new screen, you shouldn't pass it a reference to the NavController so that it can call navigate() directly.

According to Unidirectional Data Flow (UDF) principles, the composable should instead expose an event that the NavController handles. More directly put, your composable should have a parameter of type () -> Unit.

When you add destinations to your NavHost with the composable()function, pass your composable a call to NavController.navigate().

NavHost(navController = navController, startDestination = "startDestinationView") { composable(route = "routeName") { ComposableView(onNavigation = {navController.navigate(route = "routeName")}) } }
NavHost(navController = navController, startDestination = "startDestinationView") { composable(route = "routeName/{param}", arguments = listOf(navArgument("param") { type = NavType.Type}) { val param = it.arguments?.getType("param") ComposableView(param = param) } }

When you need to pass parameters from @Composable to another @Composable the next steps are required:

  1. In the departure `@Composable` add a function parameter that requires the type of value that will be sent to the next `@Composable`

    @Composable fun DepartureView( ... , onNavigation : (Any) : -> Unit) { /* A composable that uses the function */ }

    Yo can add any parameters into the de onNavigation argument

  2. In the destination `@Composable` add as a parameter the variable that represents the parameter needed in that `@Composable`

    @Composable fun DestinationView(parameterRequired : Any) { /* Use the parameter here */ }

  3. Configure the NavHost adding the starting route and the routes to navigate to another `@Composable`

    NavHost(navController = navController, startDestination = "startDestination") { composable("startDestination") { DepartureView( ... , onNavigation = { param -> navController.navigate("routeName/${param}") }) // if there is more than one parameter : composable("startDestination") { DepartureView( ... , onNavigation = { param1, param2, ... -> navController.navigate("routeName/${param1}/?${param2}") }) } composable( "routeName/{param}", arguments = listOf(navArgument(name = "param") { type = NavType.AnyType}) ) { val param = it.arguments?.getInt("param") ?: Any DestinationView(parameterRequired = param) } }

Open the Browser

To open a browser tab an open a web page, it must be done using an Intent.

Opening a Browser Tab

  1. Create `context` variable.

    val context = LocalContext.current

  2. Create an `Intent` variable and specify the function to do.

    val intent = Intent(Intent.ACTION_VIEW, Uri.parse(uri))

  3. Using the `context` created above, start a new activity specifying the`Intent`

    context.startActivity(intent)
