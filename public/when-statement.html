When Statement

The when expression in Kotlin is a powerful and versatile control structure, similar to switch statements in other languages but with more flexibility and capability. It allows you to check a value against multiple conditions or patterns, making it ideal for simplifying complex conditional statements.

Basic Syntax

The basic syntax of a when expression is as follows:

when (expression) { value1 -> // Code to execute if expression equals value1 value2 -> // Code to execute if expression equals value2 else -> // Code to execute if none of the values match }

Using when with Different Types of Conditions

Matching Exact Values

val day = 3 val dayName = when (day) { 1 -> "Monday" 2 -> "Tuesday" 3 -> "Wednesday" else -> "Invalid day" } println(dayName) // Output: Wednesday

Using Multiple Values in a Single Branch

val number = 7 when (number) { 1, 2, 3 -> println("Low number") 4, 5, 6 -> println("Mid number") 7, 8, 9 -> println("High number") else -> println("Out of range") }

Checking Ranges

val score = 85 when (score) { in 90..100 -> println("Grade: A") in 80..89 -> println("Grade: B") in 70..79 -> println("Grade: C") else -> println("Grade: F") }

Using is for Type Checking

fun identifyType(obj: Any) { when (obj) { is String -> println("It's a string of length ${obj.length}") is Int -> println("It's an integer") else -> println("Unknown type") } }

Using when as an Expression

Since when in Kotlin is an expression, it returns a value and can be assigned directly to a variable

val number = 5 val result = when (number) { in 1..10 -> "Within range" else -> "Out of range" } println(result) // Output: Within range

when and else

The else branch in a when expression acts as a default case and is executed if no other branch matches. While else is not mandatory, it is highly recommended for completeness and to handle unexpected cases.

val letter = 'X' val type = when (letter) { 'A', 'E', 'I', 'O', 'U' -> "Vowel" else -> "Consonant" } println(type) // Output: Consonant
