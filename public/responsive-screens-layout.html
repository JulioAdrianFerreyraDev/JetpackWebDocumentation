Responsive Screens Layout

There are some ways to create responsive screens:

WindowSizeClass

This class implementation allows to get the size of the device the app is running in.

There are 3 sizes:

  • Compact = Smartphone portrait

  • Medium = Tablet portrait / Fold

  • Expanded = smartphone landscape / Tablet landscape

WindowSizeClass Implementation

implementation "androidx.compose.material3:material3-window-size-class"
implementation("androidx.compose.material3:material3-window-size-class")

WindowSizeClass Use

In the activity where the responsive screen will be called, create a windowSizeClass val using calculateWindowSizeClass()

val windowSizeClass = calculateWindowSizeClass(activity = this)

The val must be inside the SetContent{ ... } brackets

Then, in your View or @Composable add a calculateWindowSizeClass parameter and pass the one created in the Activity

After that you can access to the screen sizes provided by the Class using the When scope:

when (windowSizeClass.widthSizeClass) { WindowWidthSizeClass.Expanded -> { // Add a composable or a function // that will only be shown only when the screes size matches. } WindowWidthSizeClass.Medium -> { // Add a composable or a function // that will only be shown only when the screes size matches. } WindowWidthSizeClass.Compact -> // Add a composable or a function // that will only be shown only when the screes size matches. }

You can create a @Composable that resizes according to the Device Size

when (windowSizeClass.widthSizeClass) { WindowWidthSizeClass.Expanded -> { MainButton(400.dp) WindowWidthSizeClass.Medium -> { MainButton(300.dp) } WindowWidthSizeClass.Compact -> MainButton() } @Composable fun MainButton(size: Dp = 200.dp) { OutlinedButton(onClick = { /*TODO*/ }, modifier = Modifier.width(size)) { } }

ConstraintsLayout

This way is similar as the way it was to create UI back when Java and XML. These constraints stick elements sides to a parent or another element sizes.

There are 4 constraints:

  • top-> Top,

  • end-> right,

  • bottom-> Bottom,

  • start-> Left

These constraints require that the elements to be stick with other ones, must have an ID to make the link between each others.

// creates references that work as ID val (cardRed, cardBlue, cardGreen, cardCyan, cardYellow) = createRefs()

It is also necessary to add the constraints reference/ID to the composable as follows:

Box( modifier = Modifier .size(100.dp, 200.dp) .background(Color.Red) .constrainAs(cardRed) { // Here goes the constraints links } )

Add the links to another Composable

bottom.linkTo(composableRef.bottom) start.linkTo(composableRef.start) end.linkTo(composableRef.end) top.linkTo(composableRef.top)

Add links to its parent

bottom.linkTo(parent.bottom) start.linkTo(parent.start) end.linkTo(parent.end) top.linkTo(parent.top)

By default, the constraint sticks one side to another side with no space between them. Use the margin property to add space between.

start.linkTo(parent.start, margin = 15.dp)

You can also create a chain of elements. This chain sticks vertically or horizontally the elements, and is able to change the layout space using properties similar as Space Around, Space Evenly, Space Between

createHorizontalChain( cardRed, cardBlue, chainStyle = ChainStyle.SpreadInside // Spread , SpreadInside, Packed )

Example:

The following example shows how to create a box and add constraints:

Box( modifier = Modifier .size(100.dp, 200.dp) .background(Color.Green) .constrainAs(cardGreen) { top.linkTo(cardYellow.top) end.linkTo(parent.end, margin = 15.dp) } )

You can also mix both ways WindowSizeClass and ConstraintsLayout

// Create a guideline for the parent start val startGuideCompact = createGuidelineFromStart(0.2f) val startGuideExpand = createGuidelineFromStart(270.dp) Box( modifier = Modifier .size(width = 100.dp, height = 200.dp) .background(Color.Blue) .constrainAs(cardBlue) { top.linkTo(cardCyan.bottom, margin = 20.dp) when (windowSizeClass.widthSizeClass) { WindowWidthSizeClass.Compact -> start.linkTo(startGuideCompact) WindowWidthSizeClass.Expanded -> start.linkTo(startGuideExpand) } } )
