Booleans

In Kotlin, Boolean is a primitive data type that represents a value of either true or false. It’s commonly used for conditional logic, control flow, and comparison operations. Booleans are essential for writing decision-making code in Kotlin applications.

Boolean Declaration

To declare a Boolean variable in Kotlin, you can assign true or false directly or as the result of an expression:

val isActive: Boolean = true val hasAccess = false // Kotlin infers the Boolean type

Boolean Operators

Kotlin supports several Boolean operators to manipulate and combine Boolean values.

Logical AND (&&)

The && operator evaluates to true only if both operands are true; otherwise, it returns false. This is useful when multiple conditions must be true for an action to proceed.

val isAdult = true val hasID = false val canEnter = isAdult && hasID // false, as hasID is false

Short-Circuiting: Kotlin stops evaluating as soon as it determines the result. If isAdult is false, hasID won’t be checked because the result is already false.

Logical OR (||)

The || operator returns true if at least one operand is true; otherwise, it returns false. It’s often used when only one of multiple conditions needs to be true.

val isMember = true val hasGuestPass = false val canAccess = isMember || hasGuestPass // true, as isMember is true

Short-Circuiting: If the first operand is true, the second operand isn’t evaluated since the overall result will be true anyway.

Logical NOT (!)

The ! operator, also known as logical negation, reverses the Boolean value. It turns true to false and false to true, often used to toggle states or check for a "not" condition.

Comparison Operators

Kotlin allows comparisons that result in Boolean values, essential for conditional expressions:

  • Equal to (==): Returns true if values are the same.

  • Not equal to (!=): Returns true if values are different.

  • Greater than (>) and Less than (<): Used for numerical or comparable types.

  • Greater than or equal to (>=) and Less than or equal to (<=).

Boolean Expressions in Control Flow

Boolean expressions are crucial in control flow statements such as if, when, and loops.

if Expressions

if statements use Boolean values to determine the execution path.

val isRaining = true if (isRaining) { println("Take an umbrella.") } else { println("Enjoy the sunny day.") }

when Expressions

when statements can handle Boolean conditions, though they’re more common with enums or specific values.

val hasPermission = true when (hasPermission) { true -> println("Access granted.") false -> println("Access denied.") }

Nullable Booleans

Kotlin allows Booleans to be nullable using Boolean?, meaning they can hold true, false, or null. This is useful when a Boolean might not yet have a defined state.

val isReady: Boolean? = null if (isReady == true) { println("Ready to proceed.") } else { println("Not ready.") }
