POO

Object-Oriented Programming (OOP) is a programming paradigm based on the concept of "objects," which encapsulate data ( attributes) and behavior (methods). OOP promotes modularity, code reusability, and a structured approach to software design.

Core Principles of OOP

Encapsulation

Encapsulation is the process of bundling data (attributes) and methods (functions) that operate on the data into a single unit called a class. It restricts access to some components of an object to maintain control over its state.

Access Modifiers

  • private: Accessible only within the class.

  • protected: Accessible within the class and its subclasses.

  • public: Accessible from anywhere.

  • internal: Accessible within the same module (specific to Kotlin).

Example

class BankAccount(private var balance: Double) { fun deposit(amount: Double) { if (amount > 0) balance += amount } fun withdraw(amount: Double) { if (amount > 0 && amount <= balance) balance -= amount } fun getBalance(): Double = balance }

Inheritance

Inheritance is a mechanism where a class (subclass) can inherit attributes and methods from another class (superclass), promoting code reuse.

Example:

open class Vehicle(val name: String) { open fun move() = println("$name is moving") } class Car(name: String) : Vehicle(name) { override fun move() = println("$name is driving") } class Bicycle(name: String) : Vehicle(name)

Polymorphism

Polymorphism allows objects of different classes to be treated as objects of a common supertype. It enables dynamic method binding.

Types:

  • Compile-Time Polymorphism: Achieved through method overloading (same method name, different parameters).

  • Runtime Polymorphism: Achieved through method overriding (subclass redefines a method of its parent class).

open class Animal { open fun sound() = println("Animal makes a sound") } class Dog : Animal() { override fun sound() = println("Dog barks") } class Cat : Animal() { override fun sound() = println("Cat meows") } fun makeAnimalSound(animal: Animal) { animal.sound() // Behavior depends on the actual type of `animal` }

Abstraction

Abstraction is the process of hiding the internal implementation details of a class and exposing only the functionality. This is achieved using abstract classes and interfaces.

Abstract Class

  • Cannot be instantiated.

  • May have both abstract and non-abstract methods.

Usage

  • When the classes share a "is-a" relationship (e.g., a Dog is-a Animal).

  • When you need to provide common functionality (with or without implementation) to related

abstract class Animal { abstract fun sound(): String // Abstract method open fun move() = println("Animal is moving") // Concrete method } class Dog : Animal() { override fun sound() = "Bark" } class Cat : Animal() { override fun sound() = "Meow" }

Interface Defines a contract that implementing classes must follow.

Usage

  • When a class needs to implement multiple behaviors (Kotlin supports multiple interfaces).

interface Drivable { fun drive() } interface Flyable { fun fly() } class Car : Drivable { override fun drive() = println("Car is driving") } class Plane : Drivable, Flyable { override fun drive() = println("Plane is driving on the runway") override fun fly() = println("Plane is flying") }

Key OOP Concepts

Class

A blueprint for creating objects.

class Person(val name: String, val age: Int)

Object

An instance of a class.

val person = Person("Alice", 25)

Method

A function inside a class.

class Calculator { fun add(a: Int, b: Int) = a + b }

Properties

Variables inside a class.

class Rectangle(var width: Double, var height: Double) { fun area() = width * height }

Data Classes

Special class for holding data.

data class User(val id: Int, val name: String)

Automatically generates toString, equals, hashCode, and copy methods.
